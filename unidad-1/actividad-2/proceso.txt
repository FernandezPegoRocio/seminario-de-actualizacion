<!DOCTYPE html>

<script>

class LoginForm extends HTMLElement
{
	constructor()
	{
		//Acá hay que instanciar en primer lugar (siempre) la clase base.
		super();
		
		//Hay que definir todos los miembros del elemento nuevo que queremos componer.
		this.username = document.createElement('input');
		this.password = document.createElement('input');
		this.loginBtn = document.createElement('button');

		//Preparación de las instancias
		this.username.type = 'text';
		this.password.type = 'password';
		this.username.placeholder = 'Ingrese usuario...';
		this.password.placeholder = 'Ingrese contraseña...';
		this.loginBtn.innerText = 'Iniciar sesión'

		//Agregar elementos
		this.appendChild(this.username);
		this.appendChild(this.password);
		this.appendChild(this.loginBtn);
	}

	onLoginBtnClick()
	{
		alert('Usuario inició sesión satisfactoriamente!');
	}

	connectedCallback()
	{
		console.log('Agregado al DOM');

		this.loginBtn.onclick = this.onLoginBtnClick;
	}

	disconnectCallback()
	{
		alert('Disconnect');

		this.loginBtn.onclick = null;
	}

}

customElements.define('x-loginform', LoginForm );


function main()
{
	let myCustomForm = new LoginForm();

	document.body.appendChild( myCustomForm );
}

window.onload = main;

</script>


</html>

_____________________________________________________PROCESO____________________________________________________________

La calculadora que se mostró en la actividad 1 era un elemento, el cual se encontró terminado en tres archivos también, index.html, style.css y script.js. La calculadora de la actividad 1 era una tabla HTML construida de manera estática en el propio HTML, con funciones sueltas en el JS y con estilos en el CSS, y el objetivo consistía en migrar dicho planteamiento hacia la abstracción de WebComponents.

style.css -> queda igual, no presenta modificacion.

index.html -> Se reemplaza la tabla directa por el custom element 

La principal diferencia entre ambos planteamientos es que en el planteado en la actividad 1 el HTML lo es todo, es la principal estructura y sólo el JS añade comportamiento; en cambio, en el planteado en la actividad 2, el HTML está vacío y la clase Calculadora forma, configura y se ocupa de todo el componente desde el JS. De la misma forma el HTML es solamente un punto de entrada.

Para realizar la migración se siguió el procedimiento previsto por el modelo que se proporciona:

-----Primero se ha creado la clase Calculadora extendiendo HTMLElement, invocando super(), la primera instrucción del constructor de la misma forma que en el ejemplo dado.

-----Segundo, se han indicado en el constructor todos los miembros del componente como propiedades de la propia clase: el display, 
la tabla, y cada uno de los botones (de números, operaciones, igual y borrar). Esto recuerda el ejemplo, en el que cada uno de los 
elementos que componen el formulario son como this.username, this.password, etc.

----- Tercero, todos los instanciados (tipos, textos innerText, clases CSS) cada botón obtiene la asignación.

----- En cuarto lugar, se ha montado la estructura de la tabla fila por fila incorporando cada botón 
en su respectiva celda, y por último se ha ejecutado this.appendChild(this.tabla) para introducir toda la tabla.

----- En quinto lugar quedan definidas como métodos de la clase los manejadores de eventos (onNumeroClick, 
onOperadorClick, onIgualClick, onBorrarClick) siguiendo el patrón de implementación del manejador de eventos 
onLoginBtnClick del ejemplo. Se implementó la mínima interfaz de WebComponents con connectedCallback que vincula 
todos los eventos onclick de los botones, y disconnectedCallback desvinculándoles con null, tal como se hace 
en el ejemplo de loginBtn.onclick. 

----- Y por último, se registró el componente con customElements.define('x-calculadora', Calculadora) e instanciaba desde main() 
con document.body.appendChild(Calculadora)  como para dejar el  de HTML vacío como en la prototipo. En el connectedCallback se 
quedan vinculados todos los eventos usando .bind(this); sin bind, JavaScript resuelve el this en función de quien recibe el evento, 
siendo éste el botón, siendo así que el botón no tiene las propiedades de la calculadora como el display.
Al aplicar bind(this), se logra que el this apunte siempre a la instancia de Calculadora. En el disconnectedCallback se eliminan 
todos los eventos indicándoles que son null. Los estilos originales almacenados en el style.css se conservaban, pero desde el mismo 
componente se inyectaron a través de un método _inyectarEstilos() que se encargaba de su creación.
