class LoginForm extends HTMLElement
{
	constructor()
	{
		//Acá hay que instanciar en primer lugar (siempre) la clase base.
		super();
		
		//Hay que definir todos los miembros del elemento nuevo que queremos componer.
		this.username = document.createElement('input');
		this.password = document.createElement('input');
		this.loginBtn = document.createElement('button');

		//Preparación de las instancias
		this.username.type = 'text';
		this.password.type = 'password';
		this.username.placeholder = 'Ingrese usuario...';
		this.password.placeholder = 'Ingrese contraseña...';
		this.loginBtn.innerText = 'Iniciar sesión'

		//Agregar elementos
		this.appendChild(this.username);
		this.appendChild(this.password);
		this.appendChild(this.loginBtn);
	}

	onLoginBtnClick()
	{
		alert('Usuario inició sesión satisfactoriamente!');
	}

	connectedCallback()
	{
		console.log('Agregado al DOM');

		this.loginBtn.onclick = this.onLoginBtnClick;
	}

	disconnectCallback()
	{
		alert('Disconnect');

		this.loginBtn.onclick = null;
	}

}

customElements.define('x-loginform', LoginForm );


function main()
{
	let myCustomForm = new LoginForm();

	document.body.appendChild( myCustomForm );
}

window.onload = main;

</script>


</html>

La calculadora escrita en la actividad 1, que se encontraba armada en tres archivos separados: index.html, style.css y 
script.js. La calculadora de la actividad 1 era una tabla HTML armada de forma estática en el HTML, con funciones sueltas
en el JS y con estilos en el CSS.
El objetivo fue migrar ese planteamiento hacia la abstracción de WebComponents, según el modelo propuesto por el docente y
expuesto en el ejemplo LoginForm.

style.css -> queda igual, no presenta modificacion.

index.html -> Se reemplaza la tabla directa por el custom element 

La principal diferencia entre ambos planteamientos es que en la actividad 1 el HTML es la estructura principal y el JS solo 
agrega comportamiento; en la actividad 2, en cambio, el HTML queda vacío y es la clase Calculadora la que arma, configura y 
gestiona todo el componente desde el JS. De la misma manera, el HTML es simplemente un punto de entrada.

Para llevar a cabo la migración se llevó a cabo el procedimiento propuesto por el modelo brindado:

-----Primero se construyó la clase Calculadora extendiendo HTMLElement, ejecutando super() como primera instrucción del 
constructor de la misma manera que lo hace el ejemplo presentado.

-----Segundo, dentro del constructor se han indicado todos los miembros del componente como propiedades de la clase: el 
display, la tabla, y cada uno de los botones (numéricos, operadores, igual y borrar). Esto imita el estilo del ejemplo donde 
cada elemento que conforma el formulario se considera como this.username, this.password, etc.

----- En tercer lugar, se han preparado todas las instancias: se les han asignado tipos, textos con innerText y clases CSS a
cada botón, siguiendo la misma lógica de "preparación de instancias" que se presenta en el ejemplo.

----- En cuarta instancia, se ha montado la estructura de la tabla fila por fila introduciendo cada botón en su respectiva 
celda, y finalmente se ha llevado a cabo this.appendChild(this.tabla) para añadir toda la tabla al componente de la misma 
forma que el docente realiza this.appendChild en cada campo del formulario que ha realizado.

----- Quinto, los manejadores de eventos quedan definidos como métodos de la clase (onNumeroClick, onOperadorClick, 
onIgualClick, onBorrarClick) siguiendo el patrón de implementación del manejador de eventos onLoginBtnClick del ejemplo. Se 
implementó la interfaz mínima de WebComponents con connectedCallback, que vincula todos los eventos onclick de los botones, 
y disconnectedCallback que desvincula asignándoles null, exactamente como se hace en el ejemplo con loginBtn.onclick. 

----- Y finalmente se registró el componente con customElements.define('x-calculadora', Calculadora) y se instanciaba desde
main() con document.body.appendChild(Calculadora) dejando el  del HTML vacío como en el prototipo.  En el connectedCallback se 
vinculan todos los eventos usando .bind(this). Sin bind, JavaScript resuelve el this según quien recibe el evento, que en este 
caso sería el botón, y el botón no tiene las propiedades de la calculadora como el display. Con bind(this) se garantiza que el 
this siempre apunte a la instancia de Calculadora. En el disconnectedCallback se desvinculan todos los eventos asignándoles null.
Los estilos originales guardados en el style.css se mantuvieron pero se inyectaron desde el mismo componente con un método 
_inyectarEstilos() que lo crea.
